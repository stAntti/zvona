# Enrichment pipeline и будущие AI-модули

В MVP enrichment полностью детерминирован и работает на выдуманных fixtures. Никакие сайты не сканируются, внешние API не вызываются.

## Будущий поток данных

```text
Company seed
→ проверка доступных источников
→ нормализация фактов
→ оценка достоверности
→ выявление сигналов и пробелов
→ lead score и приоритет
→ стратегия разговора
```

Каждый факт должен хранить выдуманный для demo источник, дату, confidence и статус подтверждения. Гипотезы должны визуально отличаться от фактов. Неизвестные поля превращаются в вопросы оператору.

## Контракты модулей

- `ResearchAgent` принимает компанию и возвращает enrichment-досье с фактами, источниками, сигналами и неизвестными полями.
- `SalesStrategyAgent` превращает кампанию и досье в цель, вступление, вопросы, возражения и критерии успеха.
- `LiveCopilotAgent` принимает текущий контекст разговора и возвращает подсказку, следующий вопрос, предупреждения и обновление прогресса.
- `QualityAgent` принимает транскрипт, результат и правила кампании и возвращает оценку, валидность и объяснение.
- `CampaignManagerAgent` ранжирует лиды и предлагает распределение между операторами.
- `AccountManagerAgent` отслеживает выполнение правил и риски клиентской кампании.

В коде MVP это только TypeScript-интерфейсы. Оркестрация, очереди задач, prompt management и вызовы моделей не создаются.
