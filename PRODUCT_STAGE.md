# ZVONA Product Stage

## Stage 0 — первый платящий клиент

### Цель

Получить первого платящего клиента через убедительный и целостный demo-сценарий ZVONA.

### Главный критерий решения

Перед началом любой задачи ответить:

> Почему эта задача увеличивает вероятность продажи продукта?

Задача выполняется сейчас, только если она:

- делает ценность ZVONA понятнее потенциальному клиенту;
- улучшает демонстрируемый пользовательский путь;
- устраняет видимый барьер в demo;
- позволяет быстрее провести следующий разговор или пилот.

Если связь с продажей косвенная или гипотетическая, задача отправляется в backlog.

### Приоритет

1. UX.
2. Demo.
3. Скорость.
4. Красивый интерфейс.

### Что строим сейчас

1. Карточка компании с понятным контекстом и следующими действиями.
2. AI summary, которое помогает оператору быстро понять аккаунт.
3. Очередь звонков с ясным приоритетом.
4. Экран оператора для выполнения одной следующей задачи.
5. Экран QA с понятным разбором результата.
6. Экран руководителя с прогрессом кампании и действиями на сегодня.
7. Красивый onboarding, быстро приводящий к первому полезному результату.
8. Надёжный demo-режим на выдуманных данных.

### Не делать на Stage 0

- Kubernetes;
- новые microservices;
- performance optimization;
- масштабирование;
- multi-tenancy как отдельный проект;
- advanced auth;
- security hardening без прямого блокера пользовательского сценария;
- caching;
- новые queue/job-системы;
- dependency cleanup и audit-работы;
- универсальные абстракции «на будущее».

Исключение возможно только когда проблема непосредственно блокирует текущий demo, пилот или работу с первым клиентом.

### Definition of Done для продуктовой итерации

- Изменение видно в пользовательском сценарии.
- Его ценность можно объяснить потенциальному клиенту одним предложением.
- Demo проходит без ручного объяснения сломанного или отсутствующего шага.
- Интерфейс выглядит цельно на основном desktop-размере.
- Не добавлена инфраструктура, не требуемая сценарием.

### Следующий этап

Stage 1 начинается после первого платящего клиента. Его приоритеты определяются на основе реального использования, возражений и данных пилота, а не заранее.
