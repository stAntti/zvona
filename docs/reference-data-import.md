# Импорт справочников

Пример `qa_spravochnik_2026-06-30.xlsx` содержит один лист `QnA`, 112 записей и пять колонок:

1. Вопрос;
2. Ответ;
3. Категория 1;
4. Категория 2;
5. Категория 3.

Эта структура подходит для будущего импорта базы знаний оператора в `KnowledgeBaseEntry`. Содержимое примера не переносится в ZVONA, потому что относится к другой предметной области и содержит реальные контактные данные.

Импорт в будущем должен валидировать заголовки, удалять пустые строки, отмечать дубликаты вопросов и сохранять исходные уровни категорий. В MVP внешнего импорта нет.
